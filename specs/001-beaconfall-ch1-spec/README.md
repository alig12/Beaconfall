# Beaconfall Chapter 1 Spec

This folder holds the source of truth for the Chapter 1 vertical slice of Pokémon Beaconfall.
It covers the feature spec, implementation plan, technical research, data model, quickstart
guide, and task breakdown for the first playable chapter.

## Documents

- `spec.md` - chapter goals, acceptance criteria, and player-facing flow.
- `plan.md` - implementation approach, file targets, and progression design.
- `research.md` - engine patterns and technical decisions that informed the plan.
- `data-model.md` - chapter state model, entities, and validation rules.
- `quickstart.md` - implementation order and validation checkpoints.
- `tasks.md` - task list with dependencies and validation notes.
- `checklists/requirements.md` - requirements checklist for the spec.
- `checklists/spec-plan.md` - checklist for the spec and plan pairing.

Use this directory when implementing or reviewing the Chapter 1 vertical slice.
