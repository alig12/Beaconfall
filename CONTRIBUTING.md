# Contributing to Beaconfall

Beaconfall is a new ROM hack built on `pokeemerald-expansion`. Contributions are welcome for maps,
scripting, data, docs, tooling, and engine integrations that support the game.

## Good contribution targets

- Map and event scripting
- Chapter and story content
- Trainer parties, wild encounters, and item data
- Engine work that supports Beaconfall-specific gameplay
- Docs and developer tooling

## Before you open a PR

- Check whether the change already exists in an open issue or branch.
- Keep the PR focused on one feature, fix, or docs pass when possible.
- Add screenshots or video clips for visible gameplay changes.
- Run the relevant validation steps and mention any blockers in the PR.

## Bug reports

- Open an issue in this repository.
- Include what you expected, what happened, and how to reproduce it.
- Mention your branch or commit if the bug only shows up in local work.

## Feature requests

- Describe the gameplay goal and why it fits Beaconfall.
- Link the relevant spec, note, or plan if one exists.
- Call out any map, story, or balance impact the feature would have.

## Pull requests

- Target this repo's `master` branch unless a maintainer asks otherwise.
- Include a short summary, a test plan, and any follow-up work.
- If the change affects gameplay, mention the areas you playtested.

## Credits

- If you want credit for a contribution, mention it in the issue or PR so it can be recorded.
- Engine-level work inherited from `pokeemerald-expansion` should still respect upstream credit.
